# Cornerstone Logo Masters

These SVGs are the untouched source-of-truth masters provided by the Cornerstone brand.

- `Cornerstone Logo - Icon Blk.svg` — geometric "C" icon + gold corner square (1080×1080).
- `Cornerstone Logo - Left Blk.svg` — horizontal lockup with "CORNERSTONE" wordmark (1920×300).

## Derived assets

The running web app consumes these derivatives (generated by hand; regenerate if the masters change):

- `apps/web/public/cornerstone-icon.svg` — `fill="currentColor"` version of the icon. The gold corner square stays gold; other glyphs inherit the CSS `color` of the parent element, so the icon renders correctly on both dark and light surfaces with a single asset.
- `apps/web/public/cornerstone-icon-dark.svg` — hardcoded `fill="#ffffff"` for the `<link rel="icon">` favicon slot where `currentColor` does not resolve.
- `apps/web/public/cornerstone-wordmark.svg` — same treatment applied to the full horizontal lockup.

### Composite codemark (icon + "CODE" wordmark)

`Cornerstone Codemark.svg` is a derived composite that pairs the icon with an outlined "CODE" wordmark set in Cormorant Garamond (brand-approved Calvino Grande fallback per the brand guide §7). It's used for large raster icons — macOS dock, installer, Apple touch icon — where the wordmark is legible.

The composite is regenerated (not hand-edited) from:

- `Cornerstone Logo - Icon Blk.svg` (icon master — polygons are embedded inline in the generated file)
- `fonts/CormorantGaramond.ttf` — variable serif, OFL-licensed (`fonts/OFL.txt`); the "CODE" glyphs are outlined to SVG paths so the composite is font-independent at render time.

Regeneration:

```sh
node scripts/build-cornerstone-codemark.mjs   # masters + web variants
bun scripts/build-cornerstone-favicons.mjs    # raster derivatives
```

Outputs:

- `cornerstoneLogos/Cornerstone Codemark.svg` — brand-palette master (black glyphs, gold corner).
- `apps/web/public/cornerstone-codemark.svg` — `currentColor` variant for web.
- `apps/web/public/cornerstone-codemark-dark.svg` — white-baked variant used as the raster source.

Raster favicons (`favicon-16x16.png`, `favicon-32x32.png`, `apple-touch-icon.png`, `favicon.ico`) are generated from these SVGs via `scripts/build-cornerstone-favicons.mjs` and committed to `apps/web/public/`. Sizes below 128 px fall back to the plain icon (the "CODE" wordmark would be unreadable).
